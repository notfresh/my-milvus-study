# PR #51665: fix: bitset read() range check capped multi-bit reads at 8 bits

- **Commit:** `f0bad70571e9507cab498e2ffbf88dcc25491ee2`
- **Author:** xiaofanluan <xiaofan.luan@zilliz.com>
- **Cherry-pick from:** `060be0b37177aefe7a19964ecba25bf87a13a13f`

---

## Summary

`BitsetBase::read()` 的范围检查错误地用 `sizeof(data_type)` 而非 `8 * sizeof(data_type)` 比较，导致任何大于 8 bit 的合法读取都会触发 range checker 而失败。同时缺少 bitset 结尾边界检查。修复范围检查逻辑，添加结尾边界检查，标记方法为 `const`，为 `VectorizedElementWiseBitsetPolicy` 添加缺失的 `op_read` 转发，删除从未被调用的 `BitsetBase::write()`（有同样的潜在 bug）。
